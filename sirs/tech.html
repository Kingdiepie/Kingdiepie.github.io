EQUIPMENT AND TECHNIQUES



My Worksapce - HP Pavillion Laptop

This laptop was my main workstation for the duration of my research and development. I needed a personal computer that I could use both at schol and at home with admin right. All of which are requirements that this computer fit. This Computer also left me with limitations, I was left with a dual core processor and eight gigabytes of RAM, which was enough for my script to work but required me to simplyify my numbers and calculations.

Data Source- Bureau of Labor Statistics and Other .gov Websites



All of my Data Tables with the needed information I got was from either Bureau of Labor Statistics or from a website maintained by the state I was collecting data from.

Data Storage - Google Sheets



The data for companies, taxes, households etc, are stored on google sheets. I used google sheets as opposed to excel for financial reasons, auto-save, and being stored online made the design easy.

Data Transfer - Gradle



With all my data stored in Google Sheets I needed a way to export it to google drive. I quickly found Gradle, The set-up was somewaht complicated but after initial set up it was very easy to use and adapt. Since the gradle quickstart quide exacuted in the command prompt I decided to run the program there as well.

Text Editor - ATOM



Atom is a good all around text editor and is easily modified with plug-ins to compile and auto-complete for all languages.

Programming Language - Java



I decided to use java for this project since it on of the languages I know best and it's Object Oriented style fit this project perfectly.

Program Environment - Command Prompt

Due to the gradle quickstart I decided to stick with the command prompt as the physical area where a user would execute the program. Useing the command prompt as the USer Interface came with some pros and cons. By nature the command prompt made forced all user input the be based off text instead of buttons and the like but on the bright side it allowed to to focus on the calculations themselves as opposed to the user interface. Java also makes it very easy to send outputs and receive input from to command promt.
